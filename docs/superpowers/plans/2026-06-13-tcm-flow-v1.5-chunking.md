# TCM-Flow V1.5 C0-C4 Chunking Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 基于已生成的 Evidence Tree，实现可复现、可比较的 C0-C4 五种切分策略，验证稳定 ID、来源映射和 Parent 恢复，并冻结 Chunk Manifest。

**Architecture:** 所有策略读取同一份 `evidence.jsonl`。C0/C1 作为章级通用字符切分基线，C2 按完整条文切分，C3 按结构化 EvidenceUnit 切分，C4 使用 Child 检索并恢复完整 clause Parent。所有输出必须保留来源 Evidence ID、配置和输入哈希。

**Tech Stack:** Python 3.11、Pydantic 2.13.4、PyYAML 6.0.3、langchain-text-splitters 1.1.2、标准库 `hashlib/json/statistics/unittest`。

- C0-C4 均能在真实两部古籍上生成；
- 连续运行两次，五个 JSONL 的 SHA256 完全一致；
- Chunk ID 无重复；
- 所有 Chunk 至少包含一个来源 Evidence ID；
- 所有 Chunk 不跨书、不跨篇章；
- C2-C4 不跨 clause；
- C4 每个 Child 唯一恢复到一个 clause Parent；
- Manifest 记录输入、配置和输出哈希；

- [ ] **Step 2: 实现 Evidence 装载**

实现：

```python
def load_evidence(path: Path) -> list[EvidenceUnit]:
    units = [
        EvidenceUnit.model_validate_json(line)
        for line in path.read_text(encoding="utf-8").splitlines()
        if line.strip()
    ]
    return sorted(
        units,
        key=lambda unit: (
            unit.book_id,
            unit.chapter_id,
            unit.clause_number or 0,
            unit.evidence_id,
        ),
    )
```
