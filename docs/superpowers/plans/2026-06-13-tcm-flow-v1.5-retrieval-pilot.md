# TCM-Flow V1.5 Retrieval Pilot Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 完成 140 组结构化语料人工抽检，建立 C0-C4 的 BM25、Dense、RRF 和真实 Cross-Encoder Reranker 检索平台，通过 10 条烟雾测试，并用 40 条 evidence-first 试标集验证正式检索实验的可行性。

**Architecture:** 真实索引只能在人工抽检通过并冻结 Evidence/Chunk 哈希后构建。实验索引与在线 `app/rag` 隔离：BM25 保存固定分词结果，Dense 保存按稳定 Chunk ID 排序的归一化 NumPy 向量，RRF 使用纯排名公式，Reranker 只重排 Child 检索文本；C4 在排名完成后恢复完整 clause Parent。完整语料、审核表、索引、问题集和逐题结果保存在 Git 忽略的 `data/rag_v1_5`，仓库只提交代码、配置、测试夹具、Manifest、统计摘要和实验日志。

**Tech Stack:** Python 3.10.6、Pydantic 2.13.4、PyYAML 6.0.3、NumPy 2.2.6、jieba 0.42.1、rank-bm25 0.2.2、PyTorch 2.7.0 CUDA 12.8、FlagEmbedding 1.4.0、BGE-M3、BGE Reranker v2-m3、标准库 `csv/hashlib/json/statistics/time/unittest`。

## 固定实验配置

创建 `experiments/rag_v1_5/configs/retrieval-pilot.yaml`：

```yaml
version: v1.5.0
seed: 20260612

quality_gate:
  required_status: ready
  required_reviewed_count: 140
  allowed_unresolved_critical_errors: 0

embedding:
  model: BAAI/bge-m3
  revision: 5617a9f61b028005a4858fdac845db406aefb181
  device: cuda
  use_fp16: true
  batch_size: 4
  max_length: 1024
  normalize: true

bm25:
  tokenizer: jieba
  hmm: false
  top_k: 20

dense:
  top_k: 20

rrf:
  k: 60

reranker:
  model: BAAI/bge-reranker-v2-m3
  revision: 953dc6f6f85a1b2dbfca4c34a2796e7dde08d41e
  device: cuda
  use_fp16: true
  batch_size: 2
  max_length: 1024
  candidate_k: 40
  top_k: 5
  normalize_score: true

evaluation:
  top_ks: [1, 5, 10]
  primary_granularity: clause
  smoke_min_answerable_hit_at_5: 0.70
```

固定原则：

- 不调用 V1.4 的 query rewrite、topic bonus、section bonus、关键词奖励或双路命中奖励；
- BM25 和 Dense 必须使用相同的 Chunk ID 和相同的 `text`；
- Reranker 输入为 `(query, hit.text)`；
- C4 排名完成后才将 `context_text` 暴露为恢复上下文；
- C4 的命中身份始终保留 Child `chunk_id`，不能用 Parent ID 覆盖；
- BGE-M3 和 Reranker 分阶段加载，不同时常驻 8GB 显存；
- 40 条试标仅用于检查流程和估计难度，不作为正式论文结论。

## 阶段验收标准

- 140 组审核全部完成，无未解决的 clause 边界、类型或 Parent 错误；
- Quality Gate 记录通过时的 Evidence、C0-C4 和审核文件哈希；
- C0-C4 均建立 BM25 与 Dense 索引，行号与 Chunk ID 一一对应；
- RRF 手算测试通过，公式只使用 `sum(1 / (k + rank))`；
- Reranker 产生独立相关性分数，不复用 RRF 分数；
- 10 条烟雾测试中 9 条可回答问题至少 70% 在 Hybrid + Reranker Top 5 命中 gold clause；
- C4 的 Parent 恢复成功率为 100%；
- 40 条试标满足五类各 8 条、两书各 20 条；
- 40 条全部人工审核，随机 10 条二次复核无未解决的 Evidence ID 错误；
- 输出 Recall@1/5/10、Hit@5、MRR@10、nDCG@10、延迟和索引大小；
- 报告明确标记为 pilot，不根据结果宣称方法优越。

- [ ] **Step 4: 增加审核、检索和试标 Schema**

在 `schema.py` 新增：

```python
AuditSampleType = Literal["clause", "formula", "note_or_boundary"]
AuditStatus = Literal["pending", "pass", "fail"]
ReviewDecision = Literal[
    "correct",
    "boundary_error",
    "type_error",
    "parent_error",
    "text_error",
]


class AuditRecord(BaseModel):
    model_config = ConfigDict(extra="forbid")

    audit_id: str
    book_id: str
    sample_type: AuditSampleType
    chapter_id: str
    clause_id: str
    evidence_ids: list[str] = Field(min_length=1)
    original_text: str = Field(min_length=1)
    structured_summary: str = Field(min_length=1)
    status: AuditStatus = "pending"
    decision: ReviewDecision | None = None
    reviewer: str | None = None
    reviewed_at: str | None = None
    comment: str = ""


class RetrievalHit(BaseModel):
    model_config = ConfigDict(extra="forbid")

    chunk_id: str
    strategy: ChunkStrategy
    rank: int = Field(ge=1)
    text: str
    context_text: str
    source_evidence_ids: list[str] = Field(min_length=1)
    clause_ids: list[str] = Field(min_length=1)
    retrieval_parent_id: str | None
    bm25_rank: int | None = None
    bm25_score: float | None = None
    dense_rank: int | None = None
    dense_score: float | None = None
    rrf_score: float | None = None
    reranker_score: float | None = None


QuestionType = Literal[
    "single_clause_fact",
    "formula_composition_or_use",
    "source_location",
    "multi_evidence",
    "unanswerable",
]


class PilotQuestion(BaseModel):
    model_config = ConfigDict(extra="forbid")

    question_id: str
    question: str = Field(min_length=1)
    question_type: QuestionType
    book_scope: str
    answerable: bool
    reference_answer: str
    gold_evidence_ids: list[str]
    gold_clause_ids: list[str]
    graded_relevance: dict[str, int]
    support_spans: list[str]
    review_status: Literal["draft", "approved", "rejected"]
```

约束：

- `answerable=True` 时 gold Evidence、gold clause 和 support span 均不能为空；
- `answerable=False` 时三个 gold/support 字段必须为空；
- `graded_relevance` 只允许 `1` 或 `2`；
- `review_status="approved"` 时必须填写参考答案或明确写入 `无答案`。

- [ ] **Step 3: 实现分层抽样**

公开接口：

```python
def sample_audit_records(
    evidence_units: list[EvidenceUnit],
    anomalies: list[ParseAnomaly],
    *,

) -> list[AuditRecord]:
    ...
```

规则：

1. 先按 `book_id -> chapter_id -> clause_number` 固定排序；
2. clause 样本按篇章轮转取 30 组；
3. formula 样本按篇章轮转取 20 组，优先包含显式/隐式/替代方；

5. 同一 `clause_id` 在同一 `sample_type` 中只出现一次；
6. 不足目标数量时立即抛出 `ValueError`，不能静默降低样本量。

## 执行顺序

```text
Task 1  环境、配置和契约
  ↓
Task 2  生成 140 组抽检样本
  ↓
Task 3  人工审核与 Quality Gate
  ├─ blocked → 单独修复 parser/chunker，再重建和复审
  └─ ready
       ↓
Task 4  C0-C4 BM25/Dense 索引
  ↓
Task 5  RRF 与真实 Reranker
  ↓
Task 6  检索指标
  ↓
Task 7  10 条烟雾测试
  ↓
Task 8  40 条 evidence-first 试标
  ↓
Task 9  Pilot 矩阵、报告和正式评测准入判断
```

检索代码的单元测试可以在 Task 3 人工审核期间并行开发，但 Task 4 的真实全库索引
和之后的真实运行必须等待 Quality Gate 为 `ready`。

## 时间、存储与费用

| 工作 | 预计耗时 |
| --- | ---: |
| 环境安装与模型下载 | 1-2 小时 |
| 140 组人工抽检 | 8-14 小时 |
| 错误修复与复审预留 | 4-8 小时 |
| BM25/Dense/RRF/Reranker 实现 | 1.5-2.5 个工作日 |
| 10 条烟雾测试 | 2-3 小时 |
| 40 条试标编写与审核 | 8-12 小时 |
| Pilot 运行、误差分析和报告 | 4-8 小时 |
| **合计** | **4-6 个工作日** |

存储建议：

- BGE-M3 模型仓库约 4.6 GB；
- BGE Reranker v2-m3 约 2.3 GB；
- Torch/CUDA 依赖和缓存数 GB；
- C0-C4 Dense 向量与运行结果较小；
- 建议至少预留 15 GB 可用空间。

本阶段不调用生成模型 API，API 成本为 `0`。GPU 仅用于本地 Embedding 和
Reranker 推理。
