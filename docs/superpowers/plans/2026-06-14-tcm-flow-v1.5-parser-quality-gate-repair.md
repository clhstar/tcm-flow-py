# TCM Flow V1.5 Parser Quality Gate Repair Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 修复 V1.5 语料解析质量门禁暴露的 9 个 clause 结构问题，确定性迁移未变化样本的人工审核结论，并生成仅包含受影响项的复审清单。

**Architecture:** 保持 `experiments/rag_v1_5` 离线实验边界不变。解析器将 formula marker 拆成边界起点、正文起点、名称和来源类型；审核层通过稳定语义键匹配新旧样本，仅在结构字段完全一致时继承审核字段。真实数据重建前先归档当前审计与门禁产物，重建后仍由人工复审决定 Quality Gate，代码不得自动把门禁改成 `ready`。

**Tech Stack:** Python 3.10、标准库 `re/csv/json/unittest`、Pydantic、现有 `experiments.rag_v1_5` CLI。

## Task 3: 重构 Formula Marker 并修复边界

**Files:**
- Modify: `experiments/rag_v1_5/parser.py`
- Test: `tests/rag_v1_5/test_parser.py`

**Step 1: 扩展 marker 数据结构**

将 `_FormulaMarker` 明确为：

```python
@dataclass(frozen=True)
class _FormulaMarker:
    boundary_start: int
    body_start: int
    formula_name: str | None
    original_start: int
    source_kind: str
    creates_formula: bool = True
    ingredients_without_preparation: bool = False
```

`boundary_start` 用于终止上一方，`body_start` 用于读取本方正文。

**Step 2: 分类显式、方数和冒号 marker**

增加独立正则与 helper：

```python
FORMULA_COUNT_PATTERN
FORMULA_COLON_PATTERN
_implicit_formula_name_span(...)
_formula_count_name(...)
```

规则：

- `方二/方三` 后紧接明确方名时仅作元数据，不创建 formula。
- `方二/方三` 后直接进入药物正文时，从最近独立方名或“方名+主之”恢复名称。
- `方：` 的 `boundary_start` 必须回退到真实方名开头。
- 无可靠名称时不猜名称，并记录可诊断 anomaly 或跳过该 marker。

**Step 3: 让分段使用下一 marker 的 `boundary_start`**

核心逻辑：

```python
segment_end = next_marker.boundary_start
formula_body = clause_text[marker.body_start:segment_end].strip()
```

**Step 4: 运行定向测试**

```powershell
.\.venv\Scripts\python.exe -m unittest -v tests.rag_v1_5.test_parser
```

Expected: 方数标记和隐式边界用例 GREEN；其他新增用例仍可 RED。

**Step 5: Commit**

