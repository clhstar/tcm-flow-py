# TCM-Flow V1.5 Formal-400 Retrieval Experiment Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在不复用 Smoke/Pilot 证据与问题的前提下，构建并冻结正式 400 题数据集，完成预注册的检索主实验、Chunk 对比和六组受控消融，形成论文可引用的正式检索结果。

**Architecture:** 正式实验继续放在离线目录 `experiments/rag_v1_5`，私有题目、审核表、逐题命中和运行结果只写入被 Git 忽略的 `data/rag_v1_5/formal`。先按 Evidence/Clause 分组完成 `formal_dev=200` 与 `formal_test=200` 的隔离，再编题和审核；检索参数沿用 Pilot 冻结值，不根据正式数据临时扩展搜索空间。开发集只用于管线门禁和后续回答层阈值/Prompt 开发，正式测试集在代码、矩阵和哈希冻结后只运行一次。

**Tech Stack:** Python 3.10、Pydantic 2、JSONL/CSV、PyYAML、BM25、BGE-M3、BGE Reranker、NumPy、`unittest`、本地 CUDA。

## 2. 正式数据固定配额

正式数据集共 400 题，开发集和测试集各 200 题。每个
`book × split × question_type` 单元格使用下表固定数量：

| 问题类型 | 每本书开发集 | 每本书测试集 | 两书总计 |
| --- | ---: | ---: | ---: |
| `single_clause_fact` | 30 | 30 | 120 |
| `formula_composition_or_use` | 20 | 20 | 80 |
| `source_location` | 10 | 10 | 40 |
| `multi_evidence` | 20 | 20 | 80 |
| `unanswerable` | 20 | 20 | 80 |
| **合计** | **100** | **100** | **400** |

隔离规则：

1. 先划分 Evidence Group，再编写问题。
2. 同一 clause 不能跨 `formal_dev` 和 `formal_test`。
3. 同一 answerable anchor clause 原则上只服务一道正式题。
4. `multi_evidence` 的两个以上 gold clause 必须来自同一本书和同一 split。
5. Smoke、Pilot 的 Evidence/Clause/Group/Question ID 全部排除。
6. 归一化问题文本在 Smoke、Pilot、Formal 内全局去重。
7. 正式测试集不得用于参数、Prompt、阈值或问题措辞调整。

## 3. 预注册检索矩阵

正式检索矩阵使用 14 个唯一配置。B4 与 C0 Hybrid+Reranker 共用同一运行，
P 与 C4 Hybrid+Reranker 共用同一运行，不重复计算。

| Config ID | 论文角色 | Chunk/上下文 | 检索方式 |
| --- | --- | --- | --- |
| `b1-c0-bm25` | B1 | C0 固定 500/80 | BM25 |
| `b2-c0-dense` | B2 | C0 固定 500/80 | Dense |
| `b3-c0-hybrid` | B3 | C0 固定 500/80 | BM25+Dense+RRF |
| `b4-c0-hybrid-rerank` | B4、C0 | C0 固定 500/80 | Hybrid+Reranker |
| `c1-hybrid-rerank` | Chunk 对比 | C1 递归 250/40 | Hybrid+Reranker |
| `c2-hybrid-rerank` | Chunk 对比 | C2 条文 | Hybrid+Reranker |
| `c3-hybrid-rerank` | Chunk 对比 | C3 结构感知 | Hybrid+Reranker |
| `p-c4-hybrid-rerank` | P、C4 | C4 Child 检索+Parent 恢复 | Hybrid+Reranker |
| `p-no-parent` | P - Parent | C4 Child 检索+Child 上下文 | Hybrid+Reranker |
| `p-no-structure` | P - 结构识别 | 通用 Parent-Child | Hybrid+Reranker |
| `p-no-bm25` | P - BM25 | C4 Parent 恢复 | Dense+Reranker |
| `p-no-dense` | P - Dense | C4 Parent 恢复 | BM25+Reranker |
| `p-no-reranker` | P - Reranker | C4 Parent 恢复 | Hybrid |
| `p-no-title` | P - 标题上下文 | C4 Parent 恢复但不注入书/篇/类型 | Hybrid+Reranker |
