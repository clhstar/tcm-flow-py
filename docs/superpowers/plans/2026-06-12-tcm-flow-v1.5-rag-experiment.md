# TCM-Flow V1.5 RAG Experiment Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 建立一套与现有 V1.4 在线 RAG 隔离、可复现、可审计的古籍 RAG 实验流水线，完成 C0-C4、B0-B4、P 和六组消融实验。

**Architecture:** 实验代码放在 `experiments/rag_v1_5`，原始语料、结构化语料、索引、完整评测集和运行产物放在已被 Git 忽略的 `data/rag_v1_5`。仓库只提交配置、Schema、清洗规则、少量测试夹具、语料与模型 Manifest、评测脚本和汇总结果，不提交两部古籍全文或完整评测集。

**Tech Stack:** Python 3.11、Pydantic 2、PyYAML、jieba、rank-bm25、Chroma、FlagEmbedding、NumPy、pandas、scikit-learn、DeepSeek Chat。

## 4. 人力、时间与调用预算

建议在开始前预留：

- 一名主标注者：负责 400 条问题、答案、证据和可回答性审核。
- 一名复核者：至少复核 10%，并处理争议样本。
- 解析器人工抽检：每本书至少抽查 30 条普通条文、20 个方剂和 20 个校注/断行案例。
- 标注工时：按每条 5-10 分钟估算，400 条约需 33-67 小时，另加复核和返工。
- 回答层 API：200 条测试集 × 6 个方法 × 3 次，完整矩阵约 3600 次生成；若先只跑 `B0/B4/P`，约 1800 次。
- 本地存储：至少预留 20 GB，容纳模型、五套 Chunk 索引、实验日志和中间结果。
- GPU：RTX 2070 8GB 先用小 batch 试跑；Embedding 与 Reranker 分阶段加载，不要求同时驻留显存。
