# TCM Flow V1.5 Parser Quality Gate Repair Design

## 1. 背景

V1.5 检索试验已完成人工审核和 Quality Gate 冻结。当前审核结果为：

```text
reviewed=140
pass=129
fail=11
pending=0
boundary_error=5
type_error=6
parent_error=0
text_error=0
```

Quality Gate 状态为 `blocked`，因此真实索引、Smoke-10 和 Pilot-40 均不能
继续执行。

11 条失败记录对应 9 个唯一 clause。人工已经确认：

- `jgy-chapter-25-040` 中“又方：犀角汤”独立作为第二个 `formula`；
- 当前该 clause 的 Evidence 结构符合审核口径，不再作为 parser 缺陷；
- 剩余问题使用确定性解析规则修复，不使用人工修正表、LLM 后处理或线上
  `app/rag` 逻辑。

## 2. 目标

1. 修复 9 个 clause 暴露出的通用方剂解析缺陷；
2. 保持 clause Parent、稳定来源映射和 Parent-Child 结构；
3. 防止方数标记、方名标题、药物组成和煎服法边界相互污染；
4. 重新解析并重建 C0-C4 后，只继承结构完全未变化的人工审核结论；
5. 受影响审核项重新人工复核，Quality Gate 只有在全部问题解决后才能转为
   `ready`。

## 3. 非目标

- 不引入词典式逐条修正表；
- 不使用 LLM 或嵌入模型判断方剂边界；
- 不修改原始古籍文本；
- 不修改线上 `app/rag`；
- 不改变“又方”独立 `formula` 的既定口径；
- 不在本阶段构建真实索引或运行 Pilot-40。

## 4. 已确认的根因

### 4.1 无标记方名未生成 Child

受影响 clause：

- `jgy-chapter-23-001`
- `jgy-chapter-23-002`
- `shl-chapter-10-394`

当前解析器主要依赖 `\x方名\x` 和 `方∶/方二` 标记。以下格式因此漏建
`formula/ingredients/preparation`：

```text
四时加减柴胡饮子方。
长服诃黎勒丸方（疑非仲景方。）
小柴胡汤主之……方三。
```

### 4.2 方数标记被误当成方剂起点

受影响 clause：

- `shl-chapter-09-386`
- `shl-chapter-10-394`

`方二。`、`方三。` 表示本条所附方剂数量或编次，不必然是一个独立方名。
当前规则会在 `shl-chapter-09-386` 生成幽灵 formula，并在
`shl-chapter-10-394` 漏掉实际的小柴胡汤。

### 4.3 隐式方名边界从冒号开始

受影响 clause：

- `shl-chapter-04-208`
- `shl-chapter-06-279`
- `shl-chapter-09-386`

当前隐式 marker 的 `start` 位于 `方∶`，而真实边界应位于方名开头。因此
前一个 formula 的正文、preparation 会错误包含下一个方名。

### 4.4 煎服法起点覆盖不足

受影响 clause：

- `jgy-chapter-03-002`
- `jgy-chapter-16-012`
- `jgy-chapter-23-001`

当前规则主要识别 `上二味`、`右三味`，未识别：

```text
上先……
上为末……
上各……
```

导致煎服法被并入 ingredients。

### 4.5 “方未见”被误建为 ingredients

受影响 clause：

- `jgy-chapter-19-002`

`（方未见）` 是编辑性说明。当前显式 formula 在没有煎服法起点时，会将
全部正文当作 ingredients，因而生成错误 Child。

## 5. 解析设计

### 5.1 Marker 分类

将方剂 marker 明确分为以下来源类型：

```text
explicit_markup
formula_heading
formula_count
section_boundary
generic_alternative
```

每个 marker 至少保存：

```text
boundary_start
body_start
formula_name
creates_formula
source_kind
```

- `boundary_start`：前一个 formula 必须在此位置结束；
- `body_start`：当前 formula 正文从此位置开始；
- `formula_name`：规范化后的方名；
- `creates_formula`：该 marker 是否创建 Evidence；
- `source_kind`：用于测试和排查规则优先级。

### 5.2 Marker 优先级

按以下顺序识别并消除重叠：

1. `\x...\x` 显式标记；
2. 独立方名标题；
3. `方二/方三` 方数标记；
4. `治之方/又方` 等通用标记；
5. `附方` 等只切断边界、不生成 formula 的分节标记。

高优先级 marker 覆盖同一区间时，低优先级 marker 不再创建重复 Evidence。

### 5.3 独立方名标题

标题必须满足以下约束：

- 位于行首或 clause 开头；
- 以已支持的方剂后缀或“方”结尾；
- 后面存在药物剂量行、煎服法行或“方未见”说明；
- 不能只是普通论述中的“方”字。

对以下格式提取方名：

```text
退五脏虚热，四时加减柴胡饮子方。
```

取最后一个逗号后的方名：

```text
四时加减柴胡饮子方
```

以下标题保留完整名称：

```text
长服诃黎勒丸方
```

### 5.4 方数标记

`方二。`、`方三。` 使用上下文分类：

- marker 后紧接一个或多个明确方名标题时，只作为计数元数据，
  `creates_formula=false`；
- marker 后直接进入药物组成时，创建 formula，并按以下顺序恢复方名：
  1. marker 前最近的独立方名；
  2. marker 前最近的“方名 + 主之”治疗引用；
  3. 无可靠方名时记录 anomaly，不猜测名称。

该规则要求：

- `shl-chapter-09-386` 只生成五苓散、理中丸两个 formula；
- `shl-chapter-10-394` 生成小柴胡汤 formula；
- `shl-chapter-04-208` 的 `方二` 可作为大承气汤正文起点；
- 不生成名为“方二”“方三”的 formula。

### 5.5 Formula 分段

每个 formula 的结束位置使用下一个有效 marker 的 `boundary_start`，而不是
下一个 `方∶` 冒号位置。

因此：

- 大承气汤正文不得包含“小承气汤”；
- 桂枝加芍药汤正文不得包含“桂枝加大黄汤”；
- 五苓散正文不得包含“理中丸”；
- 对应 ingredients/preparation 也不得跨入下一方。

### 5.6 Ingredients 与 Preparation

煎服法起点支持以下确定性形式：

```text
上 + 数量/味/药
右 + 数量/味/药
上先
上为
上各
```

从煎服法起点之前生成 ingredients，从起点开始生成 preparation。不存在
明确起点时，不通过语义猜测强行拆分。

### 5.7 “方未见”处理

当 formula 正文去除空白后仅包含：

```text
（方未见）
(方未见)
```

则：

- 保留 formula Evidence；
- 不生成 ingredients；
- 不生成 preparation；
- 生成 `note` Evidence，内容为 `方未见`；
- note 的 Parent 按现有 Schema 指向 clause。

### 5.8 “又方”保持不变

`jgy-chapter-25-040` 必须继续生成：

```text
formula-01：饮食中毒，烦满，治之方
formula-02：犀角汤
```

第二个 formula 不与第一个 formula 合并。

## 6. 九个 Clause 的验收结构

| clause_id | 预期结构 |
| --- | --- |
| `jgy-chapter-23-001` | 1 formula、1 ingredients、1 preparation |
| `jgy-chapter-23-002` | 1 formula、1 ingredients、1 preparation |
| `jgy-chapter-03-002` | 1 formula、1 ingredients、1 preparation |
| `jgy-chapter-16-012` | 1 formula、1 ingredients、1 preparation |
| `jgy-chapter-19-002` | 1 formula、0 ingredients、0 preparation、1 note |
| `shl-chapter-10-394` | 1 formula、1 ingredients、1 preparation，并保留“一作紧” note |
| `shl-chapter-09-386` | 2 formula、2 ingredients、2 preparation |
| `shl-chapter-04-208` | 2 formula、2 ingredients、2 preparation |
| `shl-chapter-06-279` | 2 formula、2 ingredients、2 preparation |

额外回归要求：

- `jgy-chapter-25-040` 仍为 2 formula；
- 不出现名称为 `方二`、`方三` 的 formula；
- ingredients/preparation 不包含下一方方名；
- 所有 Child 的 Parent 类型符合 Evidence Graph 约束。

## 7. ID 与可追溯性

- clause ID 不变；
- formula ID 继续按 clause 内有效 formula 顺序编号；
- ingredients/preparation ID 继续派生自 formula ID；
- 新增或删除幽灵 formula 时，受影响 clause 内的 Child ID 可以变化；
- 不受影响 clause 的 Evidence ID 和内容必须保持一致；
- 所有 Evidence 继续保留 source file、source hash 和 corpus version。

## 8. 审核迁移设计

重新解析后必须重新生成确定性 audit 样本。旧审核结果不得整表复制。

审核迁移使用语义键：

```text
(book_id, sample_type, clause_id)
```

只有以下字段在新旧样本中完全一致时，才复制：

```text
chapter_id
evidence_ids
original_text
structured_summary
```

复制字段仅限：

```text
status
decision
reviewer
reviewed_at
comment
```

规则：

- `audit_id` 是新样本中的序号标识，迁移后保留新值；
- 不做模糊文本匹配；
- 任一结构字段变化，状态重置为 `pending`；
- 新增、删除或无法唯一匹配的样本重置为 `pending`；
- 输出迁移摘要，记录 inherited/reset/missing/ambiguous 数量和 audit ID；
- 只有 pending 项需要人工复审；
- 重新执行 `review-audit` 前，不允许手工将旧 Quality Gate 改为 `ready`。

## 9. 测试策略

### 9.1 Parser 回归测试

每个根因先写失败测试，再修改生产代码：

1. 无标记金匮方名；
2. 方数标记后直接进入药物组成；
3. 方数标记后存在明确方名标题；
4. 隐式方名边界不污染上一方；
5. `上先/上为/上各` 煎服法起点；
6. `方未见` 生成 note、不生成 ingredients；
7. “又方”仍独立生成第二个 formula。

### 9.2 审核迁移测试

覆盖：

- 结构完全一致时继承审核；
- structured summary 变化时重置；
- evidence IDs 变化时重置；
- audit ID 改变但语义键和结构字段一致时仍可继承；
- 重复语义键时标记 ambiguous，不继承；
- 迁移后不可变列来自新样本。

### 9.3 全库验证

执行：

```powershell
.\.venv\Scripts\python.exe -m unittest discover -v
.\.venv\Scripts\python.exe -m compileall -q experiments\rag_v1_5 tests\rag_v1_5
.\.venv\Scripts\python.exe -m experiments.rag_v1_5.cli parse-corpus
.\.venv\Scripts\python.exe -m experiments.rag_v1_5.cli build-chunks
```

连续构建两次 C0-C4，验证输出哈希稳定。

## 10. 执行顺序

1. 归档当前 Evidence、Chunk、audit 和 Quality Gate 哈希；
2. 为 9 个 clause 增加最小回归测试并确认 RED；
3. 分根因修改 parser，每次确认 GREEN；
4. 运行完整测试和 Evidence Graph 校验；
5. 重新解析真实两书；
6. 重建 C0-C4 并验证两次哈希一致；
7. 重新生成 audit-140；
8. 迁移结构完全一致的旧审核结果；
9. 人工复审所有 reset/pending 项；
10. 重新执行 `review-audit`；
11. 只有 Quality Gate 为 `ready` 时进入真实索引阶段。

## 11. 完成标准

- 九个 clause 均满足第 6 节的结构；
- `jgy-chapter-25-040` 的两个 formula 保持不变；
- 全部单元测试和编译检查通过；
- Evidence Graph 校验通过；
- C0-C4 连续两次构建哈希一致；
- 审核迁移没有模糊继承；
- 所有 reset 项完成人工复审；
- Quality Gate 最终为 `ready`；
- 实验日志记录规则版本、统计变化、哈希变化、复审数量和最终门禁结论。
