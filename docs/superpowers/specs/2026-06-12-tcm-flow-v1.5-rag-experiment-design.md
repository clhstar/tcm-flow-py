# TCM-Flow V1.5 RAG 实验设计

> 版本定位：面向硕士论文实验的中医古籍混合检索与结构感知证据组织研究  
> 研究范围：V1.5 暂不开展多智能体对比实验，不构建知识图谱  
> 核心任务：证据检索评测 + 基于检索证据的问答生成评测

## 1. 研究背景

TCM-Flow V1.4 已实现 BM25、Dense 向量检索、结果融合、Agent Runtime、动态子智能体、澄清机制和答案 Guardrail，但当前 RAG 仍存在以下问题：

1. 数据源仅支持 Markdown，缺少古籍 TXT 的标准化处理流程；
2. 文档采用固定 `500` 字、重叠 `80` 字的统一切分方式；
3. 固定切分可能破坏古籍中的编号条文、方剂、药物组成、煎服法和校注结构；
4. BM25 与向量索引缺少统一、稳定、可复现的数据版本；
5. 当前检索效果尚未通过标准评测集和标准指标进行量化；
6. 系统缺少无答案问题评测，无法证明其能否减少错误召回和无证据回答。

因此，V1.5 不以增加更多 Agent 功能为目标，而是建立一套可复现的中医古籍 RAG 实验平台。

## 2. 研究目标

V1.5 的核心研究问题为：

> 在相同语料、Embedding、混合检索、重排模型和检索参数下，面向中医古籍条文、方剂与校注混排特征设计的结构感知 Parent-Child 证据组织方法，是否比固定长度切分获得更好的证据召回率、上下文完整性和回答忠实度？

具体研究问题如下：

- **RQ1：** RAG 是否优于不使用外部知识的 LLM 直接回答？
- **RQ2：** BM25 与 Dense 组成的混合检索是否优于单路检索？
- **RQ3：** Reranker 是否能改善前列证据排序？
- **RQ4：** 古籍结构感知切分是否优于固定长度和通用递归切分？
- **RQ5：** Parent-Child 上下文恢复是否能提高证据完整性和回答忠实度？
- **RQ6：** 完整方法是否能降低无答案问题上的错误回答率和幻觉率？

## 3. 研究边界

### 3.1 V1.5 包含

- 古籍 TXT 编码识别与 UTF-8 标准化；
- 篇章、编号条文、方剂、药物组成、煎服法和校注解析；
- 多种 Chunk 策略；
- BM25 稀疏检索；
- Dense 稠密检索；
- RRF 混合融合；
- Reranker 重排；
- Parent-Child 证据恢复；
- 400 条证据型评测数据；
- 检索层和回答层的对比实验、消融实验。

### 3.2 V1.5 不包含

- 多智能体方法对比；
- 大模型微调；
- 医疗诊断、辨证或处方推荐效果评价；
- 复杂中医知识图谱；
- OCR 模型对比研究；
- 面向公众发布原始语料、清洗语料或完整评测数据。

## 4. 方法定位与创新表述

### 4.1 混合检索定义

本文所称混合检索是：

```text
BM25 稀疏检索
    +
Dense 稠密语义检索
    +
RRF 结果融合
    +
Reranker 重排
```

混合检索不要求必须加入知识图谱。V1.5 不构建知识图谱，避免在缺少中医专家校验的条件下引入不可控的实体关系错误。

### 4.2 Parent-Child 定位

Parent-Child 是已有的层级文档检索思想，不应将其本身表述为原创方法，也不能将其称为知识图谱。

- Parent-Child 表达文档包含关系，例如“篇章 -> 条文 -> 子片段”；
- 知识图谱表达实体语义关系，例如“方剂 -> 包含 -> 药物”。

V1.5 的待验证贡献应准确表述为：

> 在已有混合检索和层级检索思想基础上，针对中医古籍中编号条文、方剂、药物组成、煎服法和校注混排的特点，设计结构感知的 Parent-Child 证据组织与上下文恢复方法。

该贡献是否成立，必须通过与固定切分强基线以及不使用古籍结构规则的通用 Parent-Child 对照进行公平实验验证，不能仅通过系统实现宣称有效。通用 Parent-Child 对照在消融实验中记为 `P - 结构识别`。

## 6. 数据清洗与结构化

原始数据处理流程：

```text
GBK 原始 TXT
    ↓
UTF-8 转换
    ↓
技术标记清理与断行合并
    ↓
篇章和编号条文解析
    ↓
方剂、药物组成、煎服法、校注识别
    ↓
结构化 JSONL
    ↓
不同 Chunk 策略生成
    ↓
BM25 和向量索引构建
```

每个结构化证据单元至少包含：

```json
{
  "book_id": "shang_han_lun",
  "book_title": "伤寒论",
  "volume": "",
  "chapter_id": "shl-taiyang-1",
  "chapter_title": "辨太阳病脉证并治上",
  "clause_id": "shl-taiyang-1-012",
  "clause_number": 12,
  "content_type": "clause",
  "parent_id": "shl-taiyang-1-012",
  "original_text": "原始条文",
  "normalized_text": "仅进行技术清理后的检索文本",
  "notes": ["校注内容"],
  "source_file": "457-伤寒论.txt",
  "source_hash": "EF2F...",
  "corpus_version": "v1.5.0"
}
```

清洗过程中应保留原始文本与检索标准化文本两份内容，确保任何处理结果都能追溯到原始文件。

## 7. Chunk 策略

Chunk 专项实验固定 Embedding、检索算法、Reranker 和 Top-K，只改变切分方式。

### C0：固定长度切分

```text
chunk_size = 500
chunk_overlap = 80
```

这是 TCM-Flow 当前实现，也是基础基线。

### C1：通用递归字符切分

```text
chunk_size = 250
chunk_overlap = 40
```

优先按段落、换行、句号和逗号切分，但不识别古籍业务结构。

### C2：编号条文切分

每个编号条文作为独立证据单元。超长条文只在条文内部继续递归切分，不跨条文拼接。

### C3：古籍结构感知切分

识别并分别处理：

- 编号条文；
- 方剂名称；
- 药物组成；
- 煎服法；
- 括号校注。

方剂组成和煎服法不得脱离其对应条文；校注单独保存，但与正文保留关联。

### C4：结构感知 Parent-Child

- Child 用于精确检索；
- Parent 用于向生成模型提供完整上下文；
- Child 命中后，根据 `parent_id` 恢复完整条文及其方剂信息；
- 不跨篇章或无关条文扩展上下文。

建议初始参数：

```text
Child：150-300 个汉字
Parent：完整编号条文，必要时控制在 500-1000 个汉字
```

最终参数只能在开发集上确定。

## 8. 索引设计

### 8.1 Dense 索引

向量化输入包含最小必要结构上下文：

```text
书名：伤寒论
篇名：辨太阳病脉证并治上
类型：条文
正文：太阳中风，阳浮而阴弱……
```

不得将模型生成的现代解释或摘要混入原始古籍向量文本。

### 8.2 BM25 索引

BM25 使用与 Dense 相同的证据单元和稳定 Chunk ID。中文分词保留方名、药名及高频古籍术语的完整形式。

### 8.3 混合融合

BM25 与 Dense 分别召回候选结果，使用 RRF 进行固定规则融合：

```text
RRF(d) = Σ 1 / (k + rank_i(d))
```

主实验中应固定 RRF 参数，避免同时研究 Chunk 和动态融合。

### 8.4 Reranker

Reranker 对混合召回的候选证据重新排序。例如：

```text
BM25 Top 20
Dense Top 20
RRF 合并去重
Reranker 重排
最终返回 Top 5
```

所有实验组使用相同候选规模和相同 Reranker。

### 8.5 索引可复现性

每次构建索引必须记录：

- 语料版本；
- 原始文件 SHA256；
- 清洗脚本版本；
- Chunk 策略和参数；
- Embedding 模型与版本；
- BM25 分词规则；
- RRF 参数；
- Reranker 模型与版本；
- 索引生成时间；
- Chunk 总数与长度统计。

## 9. 评测数据集

### 9.1 总体规模

共构建 400 条问题：

| 问题类型 | 数量 |
|---|---:|
| 单条文事实抽取 | 120 |
| 方剂组成与用法 | 80 |
| 原文定位 | 40 |
| 跨条文比较或多证据问题 | 80 |
| 无答案或证据不足问题 | 80 |
| **合计** | **400** |

《伤寒论》和《金匮要略方论》原则上各占约 50%。

### 9.2 数据划分

```text
开发集：200 条
测试集：200 条
```

先按篇章或证据条文组划分开发集和测试集，再生成问题。禁止将同一条文生成的相似问题随机分到两个集合。

开发集用于：

- Chunk 参数选择；
- Top-K 调整；
- RRF 和 Reranker 参数选择；
- Prompt 调整；
- 无答案阈值选择。

测试集在实验配置冻结后才运行。

### 9.3 非专家条件下的标注方法

评测集采用“证据先行”的构建方式：

1. 先选择明确的原文条文或方剂；
2. 模型依据原文生成问题、参考答案和证据位置；
3. 第二个模型独立检查答案是否完全受到证据支持；
4. 人工核对问题、答案和引用是否能从原文直接验证；
5. 删除需要中医诊断、辨证或疗效判断的样本；
6. 随机抽取至少 10% 样本进行第二轮人工复核。

论文中应表述为：

> 基于原文证据构建并经人工复核的评测集。

不得表述为“专家标注数据集”。

### 9.4 可纳入的问题

- 原文描述了哪些症状；
- 某方包含哪些药物；
- 某条文在何种文本条件下提到某方；
- 某句话或某方出现在哪一篇章或条文；
- 两条明确条文在文字和事实信息上有何异同；
- 口语表达是否能检索到对应古籍证据；
- 知识库没有证据时系统能否拒答。

### 9.5 不纳入的问题

- 患者应被诊断为什么证候；
- 某方是否适合现实患者；
- 应如何处方或调整剂量；
- 某种疗法是否有效；
- 需要复杂中医理论判断且不能由原文直接验证的问题。

## 10. 基线方法

基线是在公平条件下，不使用本文关键改进机制的合理参照方法。

| 编号 | 方法 | 用途 |
|---|---|---|
| B0 | LLM 直接回答，不检索 | 证明 RAG 的总体价值，仅参与回答层评测 |
| B1 | BM25 | 传统稀疏关键词检索基线 |
| B2 | Dense + 固定 `500/80` 切分 | 常规 RAG 基线 |
| B3 | BM25 + Dense + RRF + 固定 `500/80` 切分 | 标准混合检索基线 |
| B4 | B3 + Reranker | 固定切分的强混合检索基线 |
| P | 结构感知 Parent-Child + Hybrid + Reranker | 本文完整方法 |

最关键的主实验是：

```text
B4 固定切分强基线
vs.
P 结构感知 Parent-Child 完整方法
```

两组必须使用相同的 Embedding、BM25、RRF、Reranker、候选数量、最终 Top-K 和语料。唯一关键差异应是证据组织和上下文恢复方式。

## 11. 消融实验

| 消融版本 | 验证内容 |
|---|---|
| P - Parent 扩展 | Parent 上下文恢复是否改善证据完整性 |
| P - 结构识别 | 使用相同 Parent/Child 长度但不识别条文、方剂和校注，作为通用 Parent-Child 对照 |
| P - BM25 | 稀疏检索的贡献 |
| P - Dense | 稠密语义检索的贡献 |
| P - Reranker | 重排模块的贡献 |
| P - 标题上下文 | 书名、篇名和内容类型注入是否有效 |

除被删除的组件外，其余设置保持完全一致。

## 12. 评价指标

### 12.1 检索层

- **Recall@5：** 前 5 条结果覆盖相关证据的比例；
- **Hit@5：** 前 5 条结果中是否至少包含一条相关证据；
- **MRR@10：** 第一条相关证据在前 10 条中的排名质量；
- **nDCG@10：** 考虑多级相关性和排名位置的整体排序质量。

建议相关性标注：

```text
0 = 不相关
1 = 部分相关或辅助证据
2 = 直接支持参考答案的核心证据
```

### 12.2 回答层

- Exact Match 或 Token F1；
- Citation Precision；
- Citation Recall；
- 答案中被证据支持的陈述比例；
- 无答案拒答准确率；
- 无证据问题错误回答率；
- 幻觉率。

### 12.3 效率指标

- 平均检索延迟；
- 平均端到端回答延迟；
- 返回上下文字符数或 Token 数；
- 生成模型平均 Token 消耗；
- 索引大小。

效率指标用于排除“仅通过提供更多上下文获得效果提升”的情况。

## 13. 实验控制

所有对照实验必须保持：

- 相同的两部古籍；
- 相同的 400 条问题及数据划分；
- 相同的 Embedding 模型；
- 相同的生成模型；
- 相同的 Prompt；
- 相同的温度和解码参数；
- 相同的 BM25 分词规则；
- 相同的候选召回数量；
- 相同的 RRF 参数；
- 相同的 Reranker；
- 相同的最终 Top-K；
- 相同的参数调优机会。

Chunk 专项实验只改变 Chunk 策略。检索方式专项实验只改变 BM25、Dense 或 Hybrid 组合。

生成实验至少运行 3 次，报告均值和标准差；条件允许时报告 95% 置信区间。检索实验本身通常是确定性的，可在固定索引和参数下运行一次，并通过 Bootstrap 估计置信区间。

## 14. 分阶段实验顺序

### 阶段一：Chunk 策略比较

固定 Dense、Hybrid、Reranker 和 Top-K，比较 C0-C4，选出开发集上的最佳 Chunk 策略。

### 阶段二：检索方式比较

固定最佳 Chunk，比较：

```text
BM25
Dense
BM25 + Dense + RRF
BM25 + Dense + RRF + Reranker
```

### 阶段三：完整基线比较

在锁定参数后运行 B0-B4 和 P。

### 阶段四：消融实验

运行六个消融版本，分析各组件贡献。

### 阶段五：回答层评测

使用各主实验方法检索到的证据，在相同生成模型和 Prompt 下生成答案，评价事实正确性、引用质量、证据忠实度和拒答能力。

## 15. 预期可验证结论

实验结果成立时，可以支持以下结论：

1. `B2/B3 > B0`：引入外部古籍证据能够改善回答质量；
2. `B3 > B1/B2`：混合检索优于单路稀疏或稠密检索；
3. `B4 > B3`：Reranker 改善前列证据排序；
4. `P > B4`：古籍结构感知证据组织优于固定长度强基线；
5. `P > P - Parent`：Parent 恢复改善上下文完整性和回答忠实度；
6. `P > P - 结构识别`：条文、方剂和校注结构识别产生独立贡献；
7. P 在无答案问题上具有更低的错误回答率和幻觉率。

如果 P 只优于固定切分，却不能优于 `P - 结构识别` 或 B4，则不能宣称古籍结构适配机制有效，应如实将其定位为工程实现或领域应用。

## 16. 中期答辩表述

开题阶段曾计划采用知识图谱与文档检索结合的方案。中期可说明：

> 前期原型和数据分析表明，中医知识图谱中的证候、病机和诊疗关系需要高质量领域标注和专家校验。在当前缺少专家标注条件的情况下，直接使用自动抽取关系会将图谱误差引入问答过程，削弱实验结论的可信度。因此，本研究保持“混合检索增强”的总体目标不变，将具体路线聚焦为 BM25 稀疏检索、Dense 稠密检索、RRF 融合、Reranker 重排和古籍结构感知层级证据组织，并通过标准数据集和消融实验验证各组件贡献。知识图谱保留为后续扩展方向。

需要明确：

- 研究目标没有改变；
- 论文题目没有限定混合检索必须包含知识图谱；
- 调整的是实现路线，目的是减少不可控变量、提高可复现性；
- Parent-Child 是层级文档索引，不是知识图谱；
- V1.5 暂不进行多智能体对比，多智能体实验作为后续独立阶段。
