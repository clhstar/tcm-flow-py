> **2026-06-17 范围修订：** 生产集成只使用 `637-景岳全书.txt`。下文关于七本语料、

## 1. 目标

为 `tcm-flow` 增加面向成人中医内科常见症状的古籍 RAG 检索能力。系统以中医诊断学中的问诊方法组织用户信息，从经过筛选的古籍章节中检索候选证候、病机和鉴别依据，并返回可追溯的书名、卷篇、篇名和原文证据。

首版覆盖以下十个症状：

```text
头痛、眩晕、咳嗽、喘促、心悸、不寐、胃脘痛、腹痛、泄泻、便秘
```

本设计直接服务于项目集成，不建立新的论文实验矩阵，不运行 B4/P 对照实验，也不制作统计显著性报告。

## 2. 产品边界

### 2.1 系统提供

- 根据主症和已知信息生成后续问诊问题；
- 从古籍中检索与症状、证候、病机和鉴别相关的证据；
- 返回一至三个候选证候及其支持和不支持信息；
- 为每条古籍证据返回稳定引用标识和来源位置；
- 信息或证据不足时继续追问或明确表示不能形成候选辨证。

### 2.2 系统不提供

- 现代医学诊断结论；
- 中医确诊结论；
- 方剂、药物、剂量和煎服法推荐；
- 妇科、儿科、外科、针灸等专科问诊；
- 对 `TCM-Ancient-Books-master` 全库的无差别索引。

现有安全护栏继续负责健康咨询边界。古籍 RAG 只提供知识证据，不替代安全护栏或医疗风险判断。

## 4. 章节白名单

### 4.1 问诊方法白名单

《景岳全书》只纳入：

- `卷之一入集/传忠录（上）/十问篇（九）`；
- 与首版十个症状直接相关的论证章节；
- 不纳入“论列方”、妇儿科及其他纯方剂章节。

《医门法律》只纳入卷一中的：

- `一明望色之法/望色论`；
- `一明闻声之法/闻声论`；
- `一明辨息之法/辨息论`；
- `一明问病之法/问病论`；
- `一明切脉之法/切脉论`；
- `一明合色脉之法/合色脉论`。

### 4.2 症状章节白名单

章节标题按下表中的术语匹配，匹配后必须经过一次人工清单复核，最终清单以可提交的选择 manifest 为准。

| 标准症状 | 古籍标题和检索别名 |

| 咳嗽 | 咳嗽、咳逆 |
| 喘促 | 喘、喘逆、喘症、上气、短气 |
| 心悸 | 心悸、惊悸、怔忡、怔仲 |
| 不寐 | 不寐、不得卧、失眠 |
| 胃脘痛 | 胃脘痛、胃脘、心痛 |
| 腹痛 | 腹痛、腹满、心腹痛 |
| 泄泻 | 泄泻、下利 |
| 便秘 | 便秘、秘结、大便不通 |

《金匮要略方论》的辅助白名单固定为：

- `肺痿肺痈咳嗽上气病脉证治第七`；
- `胸痹心痛短气病脉证治第九`；
- `腹满寒疝宿食病脉证治第十`；
- `痰饮咳嗽病脉证并治第十二`；
- `惊悸吐衄下血胸满瘀血病脉证治第十六`；
- `呕吐哕下利病脉证治第十七`。

《黄帝内经素问》优先纳入 `移精变气论`、`诊要经终论`、`脉要精微论`、`平人气象论`、`咳论`、`举痛论` 等诊法及相关病机篇。实施时记录完整篇名，不使用模糊的全书范围。

### 4.3 内容角色过滤

每个文本单元必须标记以下角色之一：

```text
diagnostic_method
symptom_feature
syndrome_pattern
pathogenesis
differential
case
formula
dosage
preparation
```

生产检索默认只允许前六类进入候选集合。`formula`、`dosage` 和 `preparation` 可以在解析阶段识别，但不得进入首版生产索引，也不得通过 Parent 扩展重新带回回答上下文。

## 5. 数据与证据模型

外部 CP936/GBK 原文件保持只读，不修改原文。导入过程生成 UTF-8 工作副本和结构化证据，二者保留原始 SHA256、源文件名和编码信息。

每个证据单元至少包含：

```json
{
  "evidence_id": "stable-id",
  "parent_id": "stable-parent-id",
  "book_id": "jing_yue_quan_shu",
  "book_title": "景岳全书",
  "source_file": "637-景岳全书.txt",
  "volume": "卷之一入集",
  "chapter": "传忠录（上）",
  "section": "十问篇（九）",
  "original_text": "原始文本",
  "normalized_text": "只做检索规范化的文本",
  "symptom_tags": ["头痛"],
  "evidence_role": "diagnostic_method",
  "source_sha256": "..."
}
```

`original_text` 用于引用和展示，不能被规范化结果覆盖。`normalized_text` 只处理空白、兼容字符和检索别名，不做无依据的古文改写或内容修补。

选择 manifest 至少记录：

- 七个源文件的路径、编码、SHA256 和本地状态；
- 纳入和排除的卷篇清单；
- 每个篇章对应的症状与证据角色；
- UTF-8 导入产物、证据文件和索引的哈希；
- 生成时间和构建版本。

语料全文、解析证据和索引继续保留在本地 `data/` 下并由 Git 忽略；仓库只提交配置、代码和不含正文的 manifest。

## 6. 切分与索引

### 6.1 Parent-Child 组织

- Parent 是一个完整的诊法条目、症状论述或短小的证候段落；
- Child 按语义子句切分，目标上限为 300 字符；
- 超长 Parent 按原有小标题和自然段继续拆分，不跨卷篇或症状边界；
- Child 负责召回与排序，命中后恢复对应 Parent；
- 相同 Parent 的多个 Child 同时命中时只保留最高排名项；
- Parent 扩展前后都必须执行内容角色过滤。

### 6.2 生产索引

生产检索采用以下固定工程默认值：

```text
BM25 candidates = 20
Dense candidates = 20
Dense model = BAAI/bge-m3
RRF k = 60
Reranker candidates <= 40
Reranker model = BAAI/bge-reranker-v2-m3
Final evidence = Top-5
```

这些参数复用既有实验的可靠工程配置，不在本次集成中重新开展参数实验。模型 revision 必须固定在索引 manifest 中，避免运行时静默漂移。

## 7. 检索流程

```text
用户输入
  -> 识别十类主症之一
  -> 合并已经收集的问诊字段
  -> 添加古今同义词和证候鉴别词
  -> 按症状和 evidence_role 过滤语料
  -> BM25 与 Dense 并行召回
  -> RRF 融合
  -> Reranker 对 Child 排序
  -> 去重并恢复 Parent 上下文
  -> 返回 Top-5 证据及稳定引用
```

Hybrid+Reranker 作为稳健召回和排序基础。Parent 扩展用于提高上下文完整性与可引用性；本项目使用该能力不等于重新宣称 Parent-Child 整体显著优于 Hybrid+Reranker。

查询改写只允许：

- 标准化十个主症；
- 添加已经确认的问诊字段；
- 展开设计中声明的古今同义词；
- 添加候选证候之间需要鉴别的特征词。

不得在查询改写阶段预先断定证候，也不得加入用户没有提供的症状。

## 8. 问诊信息组织

十个症状共享以下基础字段：

```text
起病时间、病程、变化趋势
部位、性质、程度、发作规律
诱因、加重因素、缓解因素
伴随症状
寒热、汗、口渴、饮食
睡眠、情志、二便
既往病史、当前用药
```

症状专属字段：

| 症状 | 重点字段 |
| --- | --- |
| 头痛 | 部位、性质、时间规律、恶心、目眩、颈项不适 |
| 眩晕 | 旋转感、体位关系、恶心、耳鸣、乏力 |
| 咳嗽 | 干咳或有痰、痰色质、昼夜规律、外感诱因 |
| 喘促 | 活动或静息发作、体位、胸闷、哮鸣、咳痰 |
| 心悸 | 发作频率、持续时间、诱因、胸闷、眩晕 |
| 不寐 | 入睡困难、易醒、早醒、多梦、情志与饮食 |
| 胃脘痛 | 饮食关系、喜温喜按、胀痛或刺痛、嗳气反酸 |
| 腹痛 | 具体部位、痛势、喜按拒按、进食及排便关系 |
| 泄泻 | 次数、便质、时间规律、腹痛、饮食诱因 |
| 便秘 | 排便间隔、便质、排出困难、腹胀、口渴 |

每轮最多提出三个尚未回答的高信息量问题。问题优先用于区分当前候选证候，不重复询问已经明确的字段。

## 9. 回答与引用契约

RAG 工具返回值扩展为：

```json
{
  "status": "ok",
  "retrieval_mode": "hybrid_parent",
  "chief_symptom": "胃脘痛",
  "results": [
    {
      "citation_id": "E1",
      "content": "完整 Parent 证据",
      "matched_child": "用于排序的 Child",
      "book_title": "类证治裁",
      "volume": "卷之六",
      "chapter": "胃脘痛论治",
      "evidence_role": "syndrome_pattern",
      "evidence_id": "...",
      "parent_id": "..."
    }
  ]
}
```

回答模型只能引用本次结果中的 `E1-E5`。候选证候必须同时说明支持特征、仍缺少的鉴别信息和引用来源。没有足够证据时返回 `insufficient_evidence`，不得依靠模型参数记忆补全古籍依据。

## 10. 项目集成

集成保持现有 `retrieve_tcm_docs()` 和 `retrieve_tcm_knowledge` 工具入口，避免改变 Agent 的调用方式。内部实现从当前仅支持 `data/raw/*.md` 和固定 `500/80` 切分的轻量检索器，升级为读取预构建古籍索引的生产检索器。

主要影响边界：

- `app/rag/documents.py`：保留现有 Markdown 能力，增加结构化古籍证据加载；
- `app/rag/bm25_retriever.py`：从运行时临时构建改为读取持久化 BM25 索引；
- `app/rag/vector_store.py`：加载固定模型版本构建的古籍向量索引；
- `app/rag/retriever.py`：实现 RRF、Reranker、过滤、Parent 恢复和引用封装；
- `app/tools/builtins/retrieval_tool.py`：接受症状和问诊字段，返回新增来源元数据；
- 新增离线导入与索引构建入口，生产请求不得在运行时解析七本原始古籍。

现有 `data/raw/tcm_knowledge.md` 不删除。它作为人工整理知识源继续兼容，但必须通过 `source_type` 与古籍证据区分，古籍引用不得伪装成人工整理内容。

## 11. 异常处理

- 源文件缺失、SHA256 不匹配或 CP936 解码失败：停止构建，不生成部分索引；
- 白名单章节找不到或出现同名歧义：构建失败并列出具体文件和篇名；
- 向量模型或 Reranker 不可用：允许显式降级到 BM25，返回 `degraded=true`，不得静默降级；
- 没有满足症状和角色过滤的证据：返回 `insufficient_evidence`；
- 索引版本与 manifest 不一致：拒绝加载；
- 引用字段不完整：该结果不得进入最终 Top-5。

## 12. 最小工程验证

本集成不开展新的正式实验，只执行确保功能可用的工程验证：

1. 七个文件可以按 CP936/GBK 稳定导入，原始哈希和 UTF-8 产物哈希可追溯；
2. 章节白名单只包含确认范围，方剂、剂量和煎服法不会进入生产索引；
3. Parent、Child、来源位置和引用 ID 的关系完整且无孤儿；
4. 十个症状各执行一条固定冒烟查询，能够返回对应主题的古籍证据或明确的证据不足状态；
5. `retrieve_tcm_knowledge` 的既有调用方式继续工作；
6. 每条返回结果都能定位到书名、卷篇、篇名和原始证据；
7. 索引缺失、模型不可用和无结果路径具有明确错误或降级状态。

这些检查是代码和集成测试，不产生论文式对照、统计检验或实验结论。

## 13. 完成标准

满足以下条件即视为古籍生产 RAG 集成完成：

- 七本书的确认章节已经导入并可重复构建索引；
- 十个成人内科症状能够通过现有工具入口检索；
- 生产检索使用 Hybrid、Reranker 和受控 Parent 恢复；
- 返回内容具有稳定引用和完整来源元数据；
- 方剂、剂量和煎服法不进入首版生产检索上下文；
- 现有 Markdown 知识源和 Agent 调用契约不被破坏；
- 最小工程测试全部通过；
- 没有新增正式实验、评测矩阵或论文结果文件。
