# 面向中医问答的结构感知混合检索增强方法及公开 TCM-QG 评估

陈力恒，曹岭

重庆邮电大学 软件工程

## 摘要

中医问答系统需要在自然语言生成能力之外保证知识来源可靠、回答过程可追溯。针对大语言模型在专业问答中容易产生事实偏差和证据断裂的问题，本文基于项目 V1.6 公开 TCM-QG 实验，研究一种结构感知的混合检索增强方法。该方法在 BM25、BGE-M3 dense 检索、RRF 融合和 BGE reranker 的基础上，引入 Parent-Child 证据组织策略：先以细粒度 child chunk 进行召回与排序，再将命中的证据扩展到 parent context 提供给回答模型。本文在公开 Tianchi TCM-QG 数据上完成冻结实验，规范化后包含 18456 个问答对，其中测试集 2786 个问题。结果显示，Parent-Child 方法在答案层面相对普通 hybrid-rerank chunk 基线 B4 的优势尚不稳定，P-B4 的 Char F1 差值为 0.0033，95% 置信区间为 [-0.0033, 0.0100]；但 parent context 相比 child-only 消融具有稳定收益，P 相对 P-no-parent 的 Char F1 提升为 0.0149，citation recall 提升为 0.1350，二者 95% 置信区间下界均大于 0。人工盲审结果进一步显示，样本回答的证据支持率和引用正确率均为 74.06%，幻觉率为 0。实验说明，结构化 parent context 更适合作为提升证据覆盖与可追溯性的组件，而不宜被表述为在当前设置下整体显著优于强 hybrid-rerank 基线。

**关键词：** 中医问答；检索增强生成；混合检索；Parent-Child；证据可追溯性

## 1 引言

随着大语言模型在自然语言处理中的广泛应用，智能问答系统在语言理解、多轮交互和答案生成方面取得了明显进展。然而，在医疗和中医等专业领域，模型回答不仅要“通顺”，还必须满足事实可靠、依据清晰和风险可控等要求。中医问答尤其依赖术语、方药、证候和文献语境之间的细粒度关联，单纯依靠模型参数记忆或提示工程，容易出现知识更新滞后、生成幻觉和引用依据不透明的问题。

检索增强生成（Retrieval-Augmented Generation, RAG）通过在回答前检索外部文档，为生成模型提供显式证据，是降低专业问答幻觉的重要路径。现有 RAG 系统通常将文档切分为固定长度 chunk，再进行向量检索或混合检索。该方式有利于精确召回局部证据，但在中医问答场景中也存在一个矛盾：细粒度 chunk 便于匹配问题，却可能割裂原文上下文；较长文本能保留语义完整性，却会引入噪声并降低排序精度。因此，如何在细粒度检索能力和上层语义完整性之间取得平衡，是中医 RAG 问答系统需要解决的关键问题。

本文聚焦开题报告中“混合检索增强”方向的可验证部分，不讨论尚未在 V1.6 中实验验证的多智能体协作模块。本文贡献如下：第一，构建面向中医问答的结构感知 Parent-Child 证据组织方法，将细粒度 child 检索与 parent context 扩展结合；第二，在公开 TCM-QG 数据集上完成预注册、索引、检索、真实 LLM 回答和人工盲审的冻结实验；第三，通过普通 chunk 基线、child-only 消融和 bootstrap 统计检验，给出更克制的结论边界，为后续系统论文和学位论文实验提供可复现依据。

## 2 方法

本文方法的核心是将“检索粒度”和“回答证据粒度”解耦。对普通 chunk 基线 B4，系统将公开文档切分为 500/80 overlap 的固定片段，并在该片段集合上执行 BM25、BGE-M3 dense 检索、RRF 融合和 BGE reranker 重排序。对 Parent-Child 方法 P，系统先构建更细粒度的 child chunk 索引，用 child ranking 定位与问题最相关的局部证据，再将命中的 child 映射回对应 parent document context，去重后作为回答上下文。这样做的动机是保留 child 检索的精确匹配能力，同时让回答模型看到更完整的上层语义。

为区分 parent context 的真实贡献，实验设置了 P-no-parent 消融。P-no-parent 与 P 使用完全相同的 child ranking，但只向回答模型提供 child chunk 文本，不做 parent 扩展。因此，P 与 P-no-parent 的差异主要反映 parent context 对证据覆盖、引用和答案质量的影响。回答层还设置 B0 无证据基线，用于观察没有检索上下文时模型的行为边界。

正式检索矩阵包含 7 个配置：B1 为 BM25，B2 为 BGE-M3 dense，B3 为 BM25+dense+RRF，B4 为 hybrid+reranker，P 为 child 检索加 parent context，P-no-parent 为 child-only 消融，P-no-reranker 为移除 reranker 的诊断配置。答案层为控制成本和叙述清晰度，只运行 B0、B4、P 和 P-no-parent 四种方法。

## 3 实验设计

实验使用公开 Tianchi TCM-QG 本地源文件 `train.json`。规范化后数据集包含 18456 个问答对，其中训练池 12956 个、开发集 2714 个、测试集 2786 个。实验代码隔离在 `experiments/rag_v1_6`，大体量生成产物保留在本地 `data/rag_v1_6`，可提交 manifest 只记录哈希、数量和聚合指标，不包含原文、问题、参考答案或模型生成内容。

检索模型采用 BAAI/bge-m3 作为 dense embedding 模型，采用 BAAI/bge-reranker-v2-m3 作为重排序模型。回答模型采用 `deepseek-chat`，temperature=0，repeats=1，max_tokens=256，回答上下文 top_k=5。系统 prompt 要求模型只能依据给定公开文档证据回答，证据不足时拒答，并以 JSON 输出答案与引用。

评价指标分为三类。检索层使用 Doc Recall@5、Answer Span Hit@5 和 Answer Span Coverage@5；自动回答层使用 EM、Char F1、ROUGE-L F1、Citation Recall、Unsupported Answer Rate 和 Abstain Rate；统计检验使用以 `source_doc_id` 为重采样单位的 paired bootstrap，重采样 10000 次。人工盲审抽取 160 个主审问题，四种方法共 640 行；其中 64 行进入二审，另设 120 个问题用于 parent 消融重点审核。

## 4 实验结果

表 1 给出测试集检索结果。P 在 Doc Recall@5 和 Answer Span Hit@5 上略高于 B4，分别为 0.6360 和 0.7563；P 与 P-no-parent 共享 child ranking，因此 Doc Recall@5 相同，但 P 的 Answer Span Coverage@5 高于 P-no-parent，说明 parent context 能提升被检索证据对参考答案片段的覆盖。

| 方法 | Doc Recall@5 | Answer Span Hit@5 | Answer Span Coverage@5 |
| --- | ---: | ---: | ---: |
| B1 BM25 | 0.5919 | 0.7143 | 0.5919 |
| B2 Dense | 0.5108 | 0.6342 | 0.5107 |
| B3 Hybrid | 0.5894 | 0.7118 | 0.5893 |
| B4 Hybrid+Rerank | 0.6332 | 0.7516 | 0.6331 |
| P Parent-Child | 0.6360 | 0.7563 | 0.6360 |
| P-no-parent | 0.6360 | 0.6285 | 0.5963 |
| P-no-reranker | 0.5847 | 0.7125 | 0.5847 |

表 2 给出真实 LLM 回答的自动指标。P 的 Char F1、ROUGE-L F1、Citation Recall 均略高于 B4，Unsupported Rate 略低于 B4；但这些数值优势需要结合置信区间解释。

| 方法 | EM | Char F1 | ROUGE-L F1 | Citation R | Unsupported Rate |
| --- | ---: | ---: | ---: | ---: | ---: |
| B0 | 0.0018 | 0.2272 | 0.1705 | N/A | N/A |
| B4 | 0.2014 | 0.5892 | 0.5691 | 0.8238 | 0.1665 |
| P | 0.2060 | 0.5925 | 0.5729 | 0.8309 | 0.1569 |
| P-no-parent | 0.1809 | 0.5776 | 0.5584 | 0.6960 | 0.2886 |

Bootstrap 结果显示，B4 相对 B0 的提升显著，Char F1 差值为 0.3620，95% CI 为 [0.3458, 0.3783]。P 相对 B4 的 Char F1 差值为 0.0033，95% CI 为 [-0.0033, 0.0100]；citation recall 差值为 0.0072，95% CI 为 [-0.0032, 0.0174]。由于置信区间跨过 0，不能声称 P 在答案层面稳定优于 B4。相比之下，P 相对 P-no-parent 的 Char F1 差值为 0.0149，95% CI 为 [0.0073, 0.0226]；citation recall 差值为 0.1350，95% CI 为 [0.1206, 0.1497]；Unsupported Answer Rate 降低 0.1317。这说明 parent context 消融主张成立。

人工盲审结果与自动指标方向一致。640 行主审样本中，答案正确率为 69.38%，证据支持率为 74.06%，引用正确率为 74.06%，幻觉率为 0，临床安全问题率为 0.47%。二审样本未发现分歧。人工审核不能把 P-B4 的统计不确定性改写为强成功，但可以支持“证据支持度和引用可追溯性是该方法的主要收益”这一更稳健表述。

## 5 讨论

实验结果表明，Parent-Child 证据组织在当前公开 TCM-QG 设置中的贡献是局部而明确的：parent context 能改善 child-only 条件下的证据扩展、引用召回和 unsupported answer rate，但完整 P 方法尚未相对强 B4 hybrid-rerank 基线形成稳定答案级优势。这一结果对论文写作很重要，因为它把创新点从“整体性能显著领先”调整为“结构化证据组织提升可追溯性和证据覆盖”。

造成 P 未显著超过 B4 的可能原因有三点。第一，TCM-QG 中许多答案较短，普通 500/80 chunk 已能覆盖足够证据，parent 扩展的边际收益有限。第二，parent context 虽能补充语义背景，也可能引入与问题无关的内容，增加回答模型选择证据的难度。第三，当前 parent 扩展策略主要依赖去重和 top-k 控制，尚未加入证据压缩、句级过滤或结构化摘要机制。

因此，本文不将 V1.6 结果表述为 Parent-Child RAG 显著优于普通 chunk RAG，而将其定位为中医问答系统中证据组织模块的有效消融证据。后续工作可继续探索更短 parent、证据压缩、多智能体验证和安全审查智能体，将开题报告中的系统级协作设想落到可验证实验上。

## 6 结论

本文基于 TCM-Flow V1.6 公开 TCM-QG 冻结实验，评估了面向中医问答的结构感知混合检索增强方法。实验表明，Parent-Child 方法在当前正式测试中尚未稳定超过强 hybrid-rerank chunk 基线，但 parent context 相比 child-only 消融显著改善了 Char F1、citation recall 和 unsupported answer rate。该结果说明，结构化 parent context 对提升中医问答的证据覆盖和可追溯性具有价值，但仍需要进一步控制上下文噪声并加强证据压缩。本文为后续中医 RAG 问答系统的实验设计、论文表述和系统扩展提供了可复现依据。
