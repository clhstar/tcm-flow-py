# V1.6 公开 TCM-QG 离线代理实验结果

## 1. 公开数据集与范围

本 V1.6 运行使用公开 Tianchi TCM-QG 本地源文件 `train.json`，与早期
V1.5 Formal-400 实验隔离。Formal-400 不包含在本公开实验的结果、表格或结论中。

本次已完成的运行是离线代理实验。检索使用
`jieba_bm25_plus_overlap_rerank`；回答评估使用
`extractive_oracle_proxy`。代理回答只有在检索到的公开证据包含参考答案片段时才输出
参考答案，否则弃答。因此回答结果是检索条件下的代理/上界指标，不是真实外部 LLM
生成结果。

## 3. 方法

`B0` 是无证据基线。在离线代理实验中，因为没有提供公开证据，所以它始终弃答。

`B4` 使用普通 500/80 chunk 对公开文档进行检索。

`P` 先检索 child chunk，再把去重后的 parent document context 提供给代理回答层。

`P-no-parent` 复用与 `P` 相同的 child ranking，但只给代理回答层提供 child chunk
文本。

## 4. 检索结果

| 方法 | Doc Recall@5 | Doc MRR@10 | Answer Span Hit@5 | Answer Span Coverage@5 |
| --- | ---: | ---: | ---: | ---: |
| B4 | 0.586504 | 0.430356 | 0.711055 | 0.586471 |
| P | 0.570711 | 0.424412 | 0.702082 | 0.570711 |
| P-no-parent | 0.570711 | 0.424412 | 0.567480 | 0.516786 |

`P` 与 `P-no-parent` 共享同一 child ranking，所以 doc recall 和 MRR 相同。
`P` 通过扩展到 parent context 提高了 answer-span availability；但在这个词法后端
下，`P` 的检索排序略低于 `B4`。

## 6. 成功门控

```text
public_tcm_qg_success=false
skip_mtcmb_tcm_litdata=false
```

门控失败，因为在离线代理运行中 `P` 没有在 Char F1 上超过 `B4`。`P` 相比
`P-no-parent` 有正向置信区间，说明 parent context 在 child 检索之后有帮助；
但这不足以支撑完整预注册成功门控。

## 7. 论文表述建议

本结果只支持一个窄表述：在该公开数据集和离线代理设置下，parent expansion 相比
child-only context 能提高 child 检索结果中的 answer-span availability。
它不支持“Parent-Child 整体优于普通 chunk RAG”的说法，因为在 V1.6 词法后端下
`B4` 高于 `P`。

不要把 V1.6 公开代理结果与 V1.5 Formal-400 结果混写。也不要把本代理实验描述为
LLM 答案生成实验。

## 8. 局限

TCM-QG 篇章混合了现代和古典中医文献风格文本。结果应理解为公开文献问答验证，
不是临床建议。

回答层是确定性代理，不是随机模型。后续付费/API 模型运行需要单独预注册 V1.6
回答计划、预算确认和新的结果报告。
