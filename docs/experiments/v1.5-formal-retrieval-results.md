# TCM-Flow V1.5 Formal-400 正式检索实验结果

日期：2026-06-15  
阶段：检索层正式结果已冻结，回答生成层尚未执行

## 技术摘要

Formal-400 在冻结的 200 道正式测试题上完成 14 组检索配置，共执行 2800
次逐题检索，运行错误为 0，Top-5 可追溯率为 1.0，启用 Parent 的配置
Parent recovery rate 为 1.0。检索指标只在其中 160 道 answerable 问题上计算，
其余 40 道 unanswerable 问题用于观察检索分数分布，不参与 Recall、MRR 和 nDCG
的平均值。

主比较中，P（C4 结构感知 Parent-Child + Hybrid + Reranker）相对 B4
（C0 固定 500/80 + Hybrid + Reranker）的结果为：

- Recall@5：`+0.0063`，95% CI `[-0.0063, 0.0219]`，区间跨 0。
- MRR@10：`+0.0304`，95% CI `[0.0010, 0.0620]`，区间不含 0。
- nDCG@10：`+0.0524`，95% CI `[0.0264, 0.0793]`，区间不含 0。

因此，本实验支持“P 在该基准上改善前列排序质量”，但不支持“P 明确提高
Recall@5”。同时，P 与通用 Parent-Child 消融 `p-no-structure` 的三个区间均
包含 0，不能把 P 相对 B4 的全部收益归因于古籍结构识别本身。

本题库是经过人工审核的、片段锚定的合成古籍检索基准。结果不能直接外推为自然
用户问题、临床咨询或回答生成质量，也不能据此宣称回答正确性、忠实性或安全性已经
得到验证。

## 研究问题与结论边界

本阶段只回答检索层的 RQ2-RQ5：

- RQ2：Hybrid 检索相对单路检索的作用。
- RQ3：Reranker 的作用。
- RQ4：结构感知 Chunk 相对固定切分和通用 Parent-Child 的作用。
- RQ5：Parent recovery 的可追溯性与上下文恢复能力。

RQ1 和 RQ6 涉及最终回答质量、引用、拒答和幻觉，必须等待独立的 B0/B4/P
回答层实验。本报告不把检索命中等同于回答正确。

## 限制与稳健性边界

1. 题目是模板辅助草拟后人工审核的片段锚定合成问题，不是自然用户查询日志。
2. 320 道 answerable 问题中，规范化最长连续重合至少 8 字的有 162 道，
   至少 12 字的有 60 道，至少 20 字的有 13 道；重合比例至少 0.25 的有
   62 道。这一事实必须随论文结果披露。
3. 一审 400 题和固定分层二审 40 题已关闭，但代理辅助的 70 条 Dev 抽查不是
   独立人工复核，也不能替代外部专家评审。
4. 语料只覆盖《伤寒论》和《金匮要略》当前本地快照，不能外推到其他古籍版本、
   现代医学文本或临床实践。
5. 检索指标在 answerable 问题上计算；无答案能力尚未通过冻结阈值的分类指标验证。
6. 分层结果样本较小且未作独立多重比较控制，只能用于描述和后续误差分析。
7. Parent 消融不改变排序，因此 RQ5 的回答层收益必须通过后续生成实验验证。
8. 本结果是检索关联和工程可追溯性证据，不是诊疗有效性、医学正确性或因果证据。

这些限制不会使实验成为“学术造假”。相反，题目来源、生成方式、人工审核范围、
字面重合、私有数据边界和不能外推的结论都被明确记录。学术风险主要来自隐瞒这些
事实或把检索结果夸大为自然问答与临床效果，本报告已主动排除这种表述。

## RQ2-RQ5 判定

### RQ2：Hybrid 的作用是混合的

C0 上，Hybrid 相对 Dense 三项指标更高；相对 BM25，MRR 和 nDCG 更高，
Recall@5 更低。C4 消融中，去 BM25 只有 nDCG 区间支持正向贡献，去 Dense
的三个区间均包含 0。结论是 Hybrid 改善排序质量的证据较强，但不能宣称它在所有
指标和所有条件下都优于单路检索。

### RQ3：Reranker 提高覆盖但存在排名与延迟代价

P 相对去 Reranker 的 Recall@5 区间不含 0，但 MRR@10 delta 为负且区间不含
0，nDCG 区间包含 0。平均延迟从 32.66 ms 增至 348.20 ms。Reranker 不是
单向收益，而是覆盖、首项排名和计算成本之间的取舍。

### RQ4：结构化组织优于固定 C0 的排序证据成立，但 C4 独立贡献未确立

P 相对 B4 的 MRR 和 nDCG 配对区间不含 0，支持结构化 Chunk/Parent-Child
组合在该基准上的排序收益。但 P 与通用 Parent-Child 的三个区间均包含 0，
C3 的点估计还略高于 C4。因此不能宣称古籍结构识别本身具有已经确认的独立贡献，
也不能宣称 C4 唯一最优。

### RQ5：Parent recovery 的工程能力成立，回答层收益待验证

启用 Parent 的配置 recovery rate 为 1.0，Top-5 traceability 为 1.0；
P 与去 Parent 的检索排名完全相同。这证明实现能够稳定恢复和追溯 Parent，
但不证明 Parent 已提高回答完整性、引用质量或忠实度。

## 回答层实验入口

下一阶段应单独预注册并冻结 B0/B4/P 回答层实验：

- B0：同一生成模型在无检索证据下直接回答。
- B4：固定 C0 + Hybrid + Reranker 检索证据后生成。
- P：C4 Parent-Child + Hybrid + Reranker 检索证据后生成。
- 每题每方法至少生成 3 次，在 formal_dev 冻结 Prompt、温度和无答案阈值。
- formal_test 只运行一次，评估 Citation Precision/Recall、支持率、拒答准确率、
  幻觉率和人工偏好。
