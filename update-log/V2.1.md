## 更新说明：新增意图识别 Agent

本次更新在 TCM-Flow 的工作流中新增了 `IntentAgent`，用于在正式问诊、检索和回答生成之前，对用户输入进行结构化意图识别。该模块作为中医多智能体工作流的前置节点，负责判断用户问题属于症状咨询、原因解释、古籍条文解释、方剂知识、一般中医知识、补充问诊信息、危险信号、问候寒暄或非中医相关问题等类型，并根据识别结果给出后续路由建议。

### 主要改动

1. 新增 `IntentState` 结构化输出模型
   增加意图类型、置信度、是否属于中医领域、是否为个人健康咨询、是否存在危险信号、是否需要检索、是否进入问诊流程、路由建议等字段，使意图识别结果可以被后续工作流节点稳定使用。

2. 新增 `IntentAgent` 组件
   在 `workflow_agent/components` 中新增意图识别组件，采用“规则信号 + LLM 结构化分类 + 规则后处理”的方式完成意图判断。规则部分用于识别高确定性的关键词信号，如危险症状、古籍条文、方剂知识、原因类问题等；模型部分用于处理更复杂或隐含的语义意图；后处理部分用于确保危险信号、问候语、古籍知识等高优先级场景能够稳定路由。

3. 新增 `INTENT_SYSTEM_PROMPT`
   为 `IntentAgent` 单独设计系统提示词，明确其职责仅限于意图分类，不进行问诊、不检索、不辨证、不生成医学建议，避免前置节点越权回答用户问题。

4. 调整 LangGraph 工作流入口
   原工作流从 `InquiryAgent` 开始，现调整为从 `IntentAgent` 开始。工作流会根据 `route_hint` 决定后续路径：

   * `direct_response`：用于问候、寒暄或明显离题问题，直接生成简短回复；
   * `inquiry`：用于个人症状咨询、原因类咨询、危险信号和补充问诊信息；
   * `evidence`：用于古籍解释、方剂知识、一般中医知识等非个人问诊类问题。

5. 优化 InquiryAgent 职责边界
   原先 `InquiryAgent` 中包含部分原因类关键词判断逻辑，本次更新后将意图识别职责前移到 `IntentAgent`。`InquiryAgent` 主要负责问诊信息结构化，不再承担主要意图分类任务，使各 Agent 的职责更加清晰。

6. 增强工作流可解释性
   `IntentAgent` 的识别结果会写入 `agent_trace`，包括主意图、辅助意图、置信度、路由建议、是否需要检索、是否存在危险信号和规则命中情况，便于前端展示、调试和论文中的多智能体流程说明。

### 更新后的工作流

```text
用户输入
  ↓
IntentAgent：识别意图并决定路由
  ├── direct_response → END
  ├── evidence → EvidenceAgent → AnswerAgent → SafetyAgent
  └── inquiry → InquiryAgent → EvidenceAgent → SyndromeAgent → AnswerAgent → SafetyAgent
```

### 设计意义

本次更新使系统具备了更明确的任务分流能力。对于个人症状咨询，系统会先进入问诊信息整理流程；对于古籍、方剂和中医知识类问题，系统可以直接进入证据检索流程；对于问候或离题内容，系统可以避免不必要的检索和多智能体调用；对于胸痛、呼吸困难、黑便、呕血、意识异常等危险信号，系统能够优先识别并进入安全审查流程。

通过新增 `IntentAgent`，TCM-Flow 的多智能体架构从原来的“问诊驱动”进一步升级为“意图驱动”，增强了系统的流程可控性、任务分工清晰度和工程可扩展性，也为后续引入更复杂的动态子智能体、技能路由和安全策略提供了基础。
